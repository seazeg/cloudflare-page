# 案例模块详细计划

## 一、模块概述

案例模块用于展示公司的成功案例，包括外贸独立站、微信小程序、APP 等项目的案例展示，帮助潜在客户了解公司的服务能力和项目成果。

---

## 二、功能需求

### 核心功能

| 功能 | 说明 |
|------|------|
| 案例列表页 | 展示所有案例，支持分类筛选 |
| 案例详情页 | 展示单个案例的详细信息 |
| 首页案例展示 | 首页展示精选案例 |
| 案例分类 | 按服务类型分类（独立站/小程序/APP） |
| 数据管理 | 使用 JSON 文件管理案例数据 |

### 案例信息结构

```typescript
interface Case {
  id: string;                    // 案例ID
  title: string;                 // 案例标题
  category: '独立站' | '小程序' | 'APP';  // 分类
  client: string;                // 客户名称（可脱敏）
  industry: string;              // 行业
  cover: string;                 // 封面图
  images: string[];              // 详情图片
  description: string;           // 简短描述
  challenge: string;             // 客户痛点/挑战
  solution: string;              // 解决方案
  results: {                     // 项目成果
    metric: string;              // 指标名称
    value: string;               // 指标值
  }[];
  timeline: string;              // 项目周期
  technologies: string[];        // 使用技术
  features: string[];            // 实现功能
  testimonial?: {                // 客户评价（可选）
    content: string;
    author: string;
    position: string;
  };
  featured: boolean;             // 是否精选
  date: string;                  // 完成日期
}
```

## 七、路由规划

| 路由 | 页面 | 说明 |
|------|------|------|
| `/cases` | 案例列表页 | 展示所有案例 |
| `/cases/[id]` | 案例详情页 | 单个案例详情 |
| `/cases?category=独立站` | 筛选结果 | 按分类筛选 |

---

## 八、技术选型

| 技术 | 用途 | 说明 |
|------|------|------|
| **JSON 文件** | 数据存储 | 简单易维护，无需数据库 |
| **TypeScript** | 类型定义 | 强类型，减少错误 |
| **Next.js SSG** | 静态生成 | 构建时生成，访问快 |
| **CSS Modules** | 样式隔离 | 避免样式冲突 |
